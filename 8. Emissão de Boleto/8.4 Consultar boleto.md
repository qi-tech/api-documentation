# Consulta de Boletos

Depois de registrar os boletos, o cliente pode consultar um boleto específico utilizando a chave de identificação **BANK_SLIP_KEY**
ou consultar uma lista de boletos filtrados por parâmetros de data de registro, status, entre outros. O formato de assinatura do header
desta requisição é descrito em detalhes [aqui](?file=223).

## Consultar boleto por chave

### Request

- endpoint: /bank_slip/{BANK_SLIP_KEY}
- method: GET
- parâmetros:
 
| Parâmetro | Descrição | Exemplo |
|---|---|---|
| **`BANK_SLIP_KEY`** | Chave de indentificação do boleto. | "96b32f1a-c2bd-41a4-b4b1-a169235be68b" |

A BANK_SLIP_KEY é uma GUID que identifica o boleto e é
fornecida na resposta do [POST de instrução](?file=883).

## Consultar lista de boletos

### Request

- endpoint: /bank_slip/person/{person_key}
- method: GET
- parâmetros:
 
| Parâmetro | Descrição | Exemplo |
|---|---|---|
| **`person_key`** | Chave de indentificação do cliente. | "96b32f1a-c2bd-41a4-b4b1-a169235be68b" |
| **`bank_slip_status`** | Status do boleto | "registered" |
| **`protest_status`** | Status de protesto | "not_protested" |
| **`from_date`** | Data inicial de criação do boleto | "05-05-2020" |
| **`to_date`** | Data final de criação do boleto | "05-07-2020" |
| **`number_search`** | Nosso número (our_number) ou numero do documento(document_number) | "12345" |
